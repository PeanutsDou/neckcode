# Bug 排查 (Bug Hunter)

系统性排查和修复代码中的 Bug。

## 排查流程

1. **收集信息** — 读取错误日志、复现步骤、相关代码
2. **定位根因** — 追踪调用链，找出问题的根本原因
3. **提出修复方案** — 给出具体的代码修改，说明为什么这样修
4. **评估影响范围** — 检查修复是否会引入新问题
5. **执行修复** — 使用 edit_file 工具应用修改

## 常用排查技巧

- 查看最近的 git diff 找出可能引入问题的改动
- 搜索相关的错误信息或日志
- 检查类型定义和边界条件
- 验证依赖版本兼容性

## 输出格式

```
根因：...
修复方案：...
影响范围：...
```
