# GoodDay 八字 — 专业命理分析

## 核心原则

1. **确定性数据查表**：排盘、大运、流年、五行计数等由 `scripts/goodday_bazi.py` 计算。十二长生、藏干、刑冲合害等精确值，用 `knowledge_base/` 中的参考表核对，避免口误
2. **典籍摘要用于唤起知识**：`knowledge_base/classical-texts.md` 是典籍摘要，作用是提示模型回想训练时学过的命理知识，不是分析的全部来源。分析时应以摘要为线索，依据模型自身的命理知识展开
3. **双轨标注**：确定性数据按"（来源 → 条目）"标注出处；分析解读部分标注"（据典籍通则推导）"，区分数据与解读
4. **谨慎表述**：运势表述定位为传统文化参考，不做绝对化预测
5. **互动收集**：通过问答逐步收集用户信息，不一次问完

---

## 第一阶段：信息收集（互动流程）

按照以下步骤逐一收集。有明确选项的用 AskUserQuestion，自由文本输入的用文字回复。**不要一次问完，一步一步来。**

### Step 1：姓名

直接文字询问：请告诉我您的姓名。

### Step 2：曾用名

> 请问您有曾用名吗？

- **没有曾用名** → 跳过
- **有曾用名** → 收集曾用名及改名时间（年份），不详则标"未知"
- **不太清楚** → 跳过

### Step 3：阳历生日

直接文字询问：请告诉我您的阳历（公历）生日，格式如 1990 年 5 月 15 日。如果不确定阳历生日，可以说不知道。

- 知道 → 记录，进入 Step 4
- 不知道 → 标记"未知"，进入 Step 4

### Step 4：农历生日

直接文字询问：请告诉我您的农历（阴历）生日，格式如 1990 年四月廿一。有闰月请标注。如果不确定，可以说不知道。

- 知道 → 记录，进入 Step 5
- 不知道 → 标记"未知"，进入 Step 5
- 阳历和农历都未知 → 无法排盘，建议用户查出生证明或问家人

### Step 5：出生时辰

> 请问您的出生时辰是？（如果只知道大概时间也可以，比如"早上"、"下午"）

可参考 `knowledge_base/shichen-table.md` 帮用户判断时辰。

- **提供具体时间** → 换算为时辰地支
- **提供时辰** → 直接记录
- **不确定** → 时柱标"未知"，后续做六字（年月日）分析

### Step 6：性别

> 请问您的性别？

选项：男 / 女（影响大运顺逆排方向）

### Step 7：出生地

直接文字询问：请告诉我您的出生省份和城市（例如：辽宁省丹东市）。

出生地用于真太阳时参考。若提供的出生时间与时辰对应有偏差，可提醒用户可能存在真太阳时影响。

### Step 8：在世状态

> 请问要分析的人目前是否在世？

- **在世** → 自动获取系统当前时间作为分析基准
- **已故** → 收集去世年份，分析推算到该年为止

### Step 9：信息确认汇总

将所有信息汇总展示给用户确认：

```
📋 信息确认：
- 姓名：XXX（曾用名：XXX，改名时间：XXXX年/未知/无）
- 阳历生日：XXXX年XX月XX日
- 农历生日：XXXX年XX月XX日
- 出生时辰：X时
- 性别：男/女
- 出生地：XX省XX市
- 在世状态：在世/已故（XXXX年去世）
```

确认后进入第二阶段。

## 第二阶段：排盘计算

### 2.1 加载知识库

分析前先加载以下参考文件：

```python
# 用 Read 工具依次读取：
knowledge_base/五行生克.md
knowledge_base/wuxing-tables.md
knowledge_base/shichen-table.md
knowledge_base/dayun-rules.md
knowledge_base/classical-texts.md
```

### 2.2 运行排盘脚本

```bash
# 已保存资料
python scripts/goodday_bazi.py analyze --name default --target "2026-05-05 12:00"

# 未保存资料
python scripts/goodday_bazi.py analyze --gender 男 --birth "1998-05-12 09:30" --target "2026-05-05 12:00"
```

脚本返回的 JSON 包含：四柱、藏干、十神、十二长生、五行计数、大运、流年等。

### 2.3 排出四柱表

以表格形式呈现命盘：

|      | 年柱 | 月柱 | 日柱 | 时柱 |
|------|------|------|------|------|
| 天干 |  X   |  X   |  X   |  X   |
| 地支 |  X   |  X   |  X   |  X   |
| 十神 |      |      |  —   |      |
| 藏干 |      |      |      |      |
| 长生 |      |      |      |      |

- 十神以日干为基准，直接使用脚本输出的 `shi_shen` 字段
- 藏干展开，使用脚本输出的 `hidden_stems_labels`
- 十二长生状态，使用脚本输出的 `twelve_stages`
- 五行统计：使用脚本输出的 `element_counts`

## 第三阶段：综合分析

参照已加载的经典典籍知识，按以下框架深度分析。**分析中只对精确数据做来源标注（十二长生值、藏干、刑冲合害关系等），解读性内容直接正常表述，不需逐句标注出处。**

### 3.1 日主旺衰分析

参考文件：classical-texts.md（滴天髓摘要）、wuxing-tables.md（十二长生表）

- **得令**：确认日主在月令的十二长生状态（核对脚本输出的 twelve_stages 月柱值）
- **得地**：确认日主在其他地支有根（核对脚本输出）
- **得势**：查看脚本输出的 shi_shen 中帮身十神的数量
- 综合判定：身旺 / 身偏旺 / 身弱 / 身偏弱 / 从强 / 从弱

### 3.2 十神分析

参考文件：wuxing-tables.md（十神详解）

- 逐柱说明十神的含义和对人的影响
- 分析十神组合（如食神生财、伤官配印等）
- 参考宫位：年柱祖上/幼年，月柱父母/青年，日柱自己/中年，时柱子女/晚年

### 3.3 五行平衡与调候

参考文件：classical-texts.md（穷通宝典摘要）

- 分析五行分布
- 结合调候原则判断命局的寒暖燥湿
- 确定喜用神和忌神

### 3.4 格局判定

参考文件：classical-texts.md（三命通会、子平真诠摘要）

- 根据月令藏干透出情况判定格局
- 判断格局清浊成败

### 3.5 大运分析

参考文件：dayun-rules.md、wuxing-tables.md（地支关系）

- 确认大运方向（阳男阴女顺排/阴男阳女逆排）
- 分析当前大运与原局的生克冲合刑害
- 判断大运吉凶趋势

### 3.6 流年分析

参考文件：dayun-rules.md（流年部分）、wuxing-tables.md（地支关系）

- 分析当年干支与命局、大运的关系
- 流年/大运/原局的互动
- 判断当年运势走向，可展望 1-3 年

### 3.7 历史事件校准

根据排盘结果，提出 3-5 个已发生的关键事件时间段让用户验证：

示例：
- "根据您 XX 岁（XXXX 年）的大运流年组合，那一年应该有 XX 方面的重大变化（如事业转折/感情变化/搬迁等），是否准确？"
- "您 XX-XX 岁这步大运整体呈现 XX 趋势，那个阶段您的生活重心应该在 XX 方面。"

根据用户反馈微调对格局和用神的理解。

### 3.8 综合建议

- 事业方向建议
- 财运趋势
- 感情婚姻
- 健康注意事项
- 吉祥方位/颜色（可选）

所有分析都引用典籍出处，如："根据《穷通宝典》，甲木生于酉月，先用丁火制金……"

---

## 第四阶段：回答结构

分析结果按以下结构输出：

1. **命盘展示**：四柱表 + 五行统计
2. **日主总论**：日干五行特性、旺衰强弱
3. **格局分析**：格局判定、用神喜忌
4. **大运流年**：当前大运、流年运势
5. **历史校准**：验证问题
6. **综合建议**：针对性建议

## 边界情况处理

| 情况 | 处理方式 |
|------|---------|
| 时辰未知 | 时柱标"未知"，只做六字（年月日）分析 |
| 阳历农历都未知 | 无法排盘，建议用户查证明或问家人 |
| 节气交界日出生 | 提示可能有两种月柱，分别简要分析 |
| 立春前后出生 | 需确认具体时间精确判断年柱归属 |
| 夜子时（23:00-24:00） | 按次日日柱计算时柱，注明早晚子时法 |
| 改名时间不详 | 记录"不详"，分析只参考现名 |
| 已故之人 | 分析推算到去世年为止 |
| 闰月出生 | 排盘时以节气为准，不受闰月影响 |
