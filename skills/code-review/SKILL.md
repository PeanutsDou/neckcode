# 代码审查 (Code Review)

对指定文件或目录进行全面的代码审查。

## 检查维度

1. **正确性** — 逻辑错误、边界条件、类型安全
2. **安全性** — 注入风险、敏感信息泄露、权限问题
3. **性能** — 不必要的循环/IO、内存泄漏、算法复杂度
4. **可维护性** — 命名清晰度、函数职责单一、重复代码
5. **风格** — TypeScript strict mode 遵守情况、项目风格一致

## 输出格式

对每个问题标注严重程度：
- **Critical** — 会导致崩溃或安全漏洞，必须修复
- **Warning** — 潜在问题，建议修复
- **Info** — 风格或可读性改进建议

## 执行方式

1. 读取目标文件
2. 逐项检查每个维度
3. 汇总所有发现的问题
4. 给出整体评分和改进建议
