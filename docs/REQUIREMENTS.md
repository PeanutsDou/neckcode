# Neck Code — 需求文档

## 一、项目定位

自建 Agent 运行时，构建一个**桌面 GUI 编码助手**。不包 claude.exe 的壳，而是在 `D:\douzhongjun\about\learn-dou\CC\` 源码基础上，自行控制 Agent 循环、Provider 层、会话管理，前端用 Electron 做桌面壳。

解决的核心问题：

- CLI 下跨 Provider 切模型（如 Claude → DeepSeek）需要改配置重启；同 Provider 内虽可 `/model` 切换，但无法多 Provider 共存、随时切换。GUI 下所有 Provider 的模型统一管理，会话内热切换，不中断对话
- CLI 纯文本界面无法常驻展示可交互的目录树、Diff 对比、图片等富内容——即使能输出文本形式的目录/代码，也是瞬态、不可交互、滚走即丢失的
- CLI 的 stdin/stdout 接口限制多模态输入（图片/音频/视频实体，而非路径）
- CLI 多 Provider（DeepSeek、千问、Ollama、Claude 等）支持不便
- CLI 工具调用结果淹没在文本流中，需要结构化展示
- CLI 会话随终端关闭而销毁，历史对话、项目上下文、Agent 记忆无法复用；每次打开终端都从零开始，不同项目的多个任务无法持久化保留和明确切换。GUI 下需做到：会话持久化存储，关闭窗口不丢上下文；多会话独立管理，显式切换，随时恢复之前的工作流和决策链

参考产品形态：**Codex 桌面版**（独立窗口、侧边栏对话、内嵌编辑器/查看器）。

## 三、功能需求

### F1. 桌面窗口壳

**优先级**: P0

- 独立桌面窗口，Windows 10/11 优先
- 窗口标题栏显示当前项目名 + 会话名
- 系统托盘常驻，支持最小化到托盘
- 窗口尺寸/位置/布局记忆，重启恢复
- 支持多窗口（每个窗口 = 一个独立的后端会话）

### F2. 对话面板（核心交互区）

**优先级**: P0

- 左侧对话面板（可拖拽调整宽度，可折叠）
- 对话气泡式展示：
  - 用户消息气泡（含所附图片/文件缩略图）
  - AI 回复气泡（流式渲染）
  - 工具调用卡片（可折叠）
  - 工具结果卡片
  - 系统消息（上下文压缩、会话切换等）
- AI 回复支持：
  - **流式输出**：逐 token 渲染，可中断
  - **富文本**：Markdown + 代码语法高亮 + 表格 + 图片内联
  - **Diff 预览**：内联展示文件修改（unified / side-by-side）
  - **文件引用**：点击路径直接打开到编辑器对应位置
- 消息操作：复制、重新生成、编辑后重发、删除
- 会话历史持久化（SQLite），重启恢复完整对话

### F3. 代码查看/编辑面板

**优先级**: P0

- 右侧主区域：代码编辑器
  - Monaco Editor（VS Code 内核）
  - 语法高亮、代码折叠、minimap
  - 多 Tab 打开文件
  - Ctrl+S 保存、Ctrl+Z/Y 撤销重做
  - Diff 编辑器模式（AI 修改后预览）
- 左侧文件树：
  - 工作区目录结构浏览
  - 单击打开、双击重命名
  - 右键菜单：新建/删除/重命名/复制路径
  - 拖拽移动文件
  - 文件变更标记（Git 感知，P2）
- 编辑器与对话联动：
  - 选中代码 → 右键"发送到对话"
  - AI diff → 一键 Apply / Reject
  - AI 修改文件后编辑器自动刷新

### F4. 多模型 + 多 Provider 支持

**优先级**: P0

后端自建 Provider 抽象层，支持：

- **OpenAI 兼容**：DeepSeek、通义千问、硅基流动、Ollama、vLLM 等
- **Anthropic 官方 SDK**：Claude 系列
- Provider 热管理：
  - 设置界面添加/编辑/删除 Provider
  - API Key 加密存储（系统凭证管理器 / AES）
  - 每个 Provider 下可配置多个模型
- 会话级模型热切换：
  - 顶部下拉框切换模型
  - **同会话内切换不中断会话**，切换后的消息走新模型
- 模型对比模式（P1）：
  - 同时发给 2-3 个模型
  - 并排对比结果
  - 可采纳任一结果

### F5. 多模态输入

**优先级**: P1

后端自行构造 API 请求体，不受 CLI 限制，支持：

- **文本**（P0）：多行输入框，Shift+Enter 换行，Enter 发送
- **图片**（P1）：
  - 粘贴（剪贴板）/ 拖拽 / 按钮上传
  - 缩略图预览，发送前可移除
  - jpg/png/gif/webp，自动压缩大图
  - 以 base64 / URL 注入 API 请求（vision 模型）
- **文件**（P1）：
  - 拖拽文件到输入框
  - 直接读取文件内容注入上下文（非路径引用）
  - 支持文本文件、PDF、Office 文档解析
- **截图**（P2）：框选屏幕区域，直接发送
- **语音**（P2）：麦克风录音 → 识别 → 填入输入框

### F6. 多模态输出

**优先级**: P1

- **富文本**（P0）：Markdown + 代码高亮 + 表格 + LaTeX
- **图片**（P1）：API 返回的 base64 / URL 图片内联渲染
- **结构化 Diff**（P1）：side-by-side + unified 模式
- **Mermaid / 流程图**（P2）：代码块自动渲染
- **语音播报**（P3）：TTS 朗读，可关闭

### F7. 工具调用可视化

**优先级**: P0

- 工具调用以可折叠卡片展示（非原始 JSON）
- 不同工具用不同图标和颜色：

| 工具       | 可视化                |
| -------- | ------------------ |
| 读文件      | 文件路径 + 代码预览（语法高亮）  |
| 写文件      | 文件路径 + diff 对比     |
| Shell 命令 | 终端风格 stdout/stderr |
| 搜索       | 结果列表，点击跳转          |
| Web 请求   | URL + 响应摘要         |

- 执行中 loading 动画
- 执行失败红色高亮，点击展开错误详情
- 危险操作（删除文件、force push 等）弹确认框

### F8. 会话管理

**优先级**: P0

- 侧边栏会话列表：
  - 按项目分组
  - 自动标题（首条消息摘要）+ 可手动重命名
  - 最后活跃时间
- 操作：新建、切换、重命名、删除、导出（Markdown / JSON）
- 多会话并行（每会话独立上下文，后端一个进程单线程调度）
- 自动保存每轮对话到 SQLite
- 重启后完整恢复

### F9. Slash 命令与 Skill 系统

**优先级**: P1

- 输入框 `/` 触发命令面板
- 可注册可扩展
- 内置命令：`/help` `/clear` `/model` `/config` `/review` `/commit` `/file`
- Skill 从本地目录热加载

### F10. 上下文管理可视化

**优先级**: P1

- Token 用量进度条（当前 / 模型上限）
- 接近上限黄色提醒、超限红色警告
- 自动压缩/摘要（compaction）
- 手动勾选排除某条消息

### F11. 配置与设置

**优先级**: P0

- 设置面板（侧边或独立窗口）：
  - Provider 管理（增删改，Key 加密存储）
  - Agent 参数（maxTurns、shellTimeout、workspaceRoot）
  - 界面（主题/字体/语言）
  - 快捷键自定义
  - 自定义 System Prompt
- 配置 JSON 本地持久化
- 导入/导出

### F12. 内嵌终端

**优先级**: P1

- 可选显示/隐藏的终端面板
- AI 命令实时输出
- 用户可手动输入
- 终端输出可发送到对话

## 四、非功能需求

### N1. 性能

- 流式 token 到渲染延迟 < 50ms
- 启动 < 3 秒
- 大文件（>1MB）编辑器不卡顿
- 长对话（>100 轮）虚拟滚动，不卡顿
- 单进程内存 < 500MB（含 1 个模型连接）

### N2. 安全性

- API Key 加密存储（系统凭证管理器 / AES-256-GCM），禁止明文落盘
- Shell 危险命令确认弹框
- 文件操作限制在工作区目录内
- API Key 不记录到日志

### N3. 可靠性

- 网络断开自动重连
- 流式中断可重试
- 崩溃后恢复未保存草稿

### N4. 可扩展性

- Provider 接口化：新增模型供应商只需实现一个接口
- Tool 可注册/扩展
- Skill 热加载
- 主题可自定义

### N5. 兼容性

- Windows 10/11 优先
- 后续 macOS、Linux
- 不依赖 WSL

## 五、技术约束（已确认）

| 决策项        | 结论                                                            |
| ---------- | ------------------------------------------------------------- |
| 架构模式       | 自建 Agent 运行时，不包 claude.exe                                    |
| 源码基础       | `CC/app/`（runtime + tools + providers）+ `CC/src/`（React 组件参考） |
| 桌面框架       | Electron（React + Monaco Editor + Node.js 后端同进程）               |
| 前后端通信      | Electron IPC（主进程 Agent → 渲染进程 UI）                             |
| Agent 运行时  | Node.js，复用 `app/agent/runtime.js` 的 ReAct 循环                  |
| Provider 层 | 统一抽象接口，支持 OpenAI 兼容 + Anthropic SDK                           |
| 数据存储       | SQLite（会话/消息/配置）+ JSON（配置文件）                                  |
| 编辑器        | Monaco Editor                                                 |
| 构建工具       | Vite（Renderer）+ esbuild/tsc（Main）                             |
