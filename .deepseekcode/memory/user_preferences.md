# 用户偏好

## 编码风格

- 偏好简洁直接的实现，不过度抽象
- 不写无意义的注释，代码自解释
- 类型安全优先（TypeScript strict mode）
- 变量/函数命名清晰即可，不追求过度语义化

## UI 偏好

- 浅色主题，暖色调，素雅风格
- 不喜欢过多的边框和分割线，用明暗差区分
- 按钮用文字而非 emoji
- 界面简洁，功能可折叠/展开
- 布局：左侧会话列表（常驻）、中间聊天、右侧代码面板（可收起）

## 工作习惯

- 喜欢一次性把功能做完再测试
- 每个模块完成后自己测试跑通
- 偏好 DeepSeek v4-pro 作为主力模型
- 关注上下文用量，避免浪费 token
